# 結合テスト設計書

## 1. テスト環境
- アプリケーションを起動した状態でのブラウザ操作または HTTP リクエストによるテスト。

## 2. テストケース
| No | 項目 | 手順 | 期待値 |
|----|------|------|--------|
| 1 | ISBN検索から登録まで | ISBN入力 -> 確認 -> 読書日入力 -> 登録 | DBにデータが入り、一覧に表示されること |
| 2 | openbd連携 | 実在するISBNを入力 | 正しい本情報（タイトル等）が画面に引き継がれること |
| 3 | 画面遷移 | 登録後に一覧へ遷移 | URLが `/books/list` になり、登録した本が見えること |
| 4 | 読了日の更新 | 一覧から更新ボタン押下 | DBの `end_date` が更新され、画面表示も変わること |

## 3. 確認方法
- 画面上の表示確認。
- H2 Console等によるDBの状態確認。
