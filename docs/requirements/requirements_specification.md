# 要件定義書

## 1. 目的
ユーザーが読んだ本、または読みかけの本を管理するための本棚アプリケーション。ISBNコードを利用して簡単に本を登録できることを目的とする。

## 2. 業務フロー
1. ユーザーは本のISBNコードを入力する。
2. システムはISBNコードを元に本情報を取得（openbd API）し、確認画面に表示する。
3. ユーザーは読み始めた日、読み終えた日を入力して登録する。
4. システムは登録された本を一覧画面に表示する。

## 3. 機能要件
- **ISBN入力機能**: ISBNコードを入力し、本情報を検索する。
- **本情報取得機能**: openbd APIからタイトル、著者、出版社、発売日、カバー画像を取得する。
- **本登録機能**: 本の情報に加えて、読書開始日、読書終了日を登録する。
- **本一覧表示機能**: 登録済みの本を一覧で表示する。
- **本編集機能**: 読み終えた日を後から更新できる。

## 4. 非機能要件
- **ユーザビリティ**: 日付入力にはカレンダーUI（JavaScript）を使用する。
- **メンテナンス性**: クリーンアーキテクチャを採用し、テスト容易性を確保する。
- **拡張性**: 異なる本情報APIへの差し替えが容易な設計にする。
