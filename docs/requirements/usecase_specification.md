# ユースケース仕様書

## UC1: 本を検索する
- **概要**: ISBNコードから本情報を取得する。
- **アクター**: ユーザー
- **事前条件**: なし
- **事後条件**: 本情報が表示される。
- **基本フロー**:
    1. ユーザーがISBNコードを入力する。
    2. ユーザーが確認ボタンを押す。
    3. システムがopenbd APIへ問い合わせる。
    4. システムが取得した情報を表示する。

## UC2: 本を登録する
- **概要**: 本の情報と読書記録を保存する。
- **アクター**: ユーザー
- **事前条件**: UC1が完了していること。
- **事後条件**: 本と記録がDBに保存される。
- **基本フロー**:
    1. ユーザーが読書開始日、終了日を入力する。
    2. ユーザーが登録ボタンを押す。
    3. システムが情報を保存する。

## UC3: 本の一覧を見る
- **概要**: 登録済みの本の一覧を表示する。
- **アクター**: ユーザー
- **事前条件**: なし
- **事後条件**: 登録済みの一覧が表示される。

## UC4: 読了日を更新する
- **概要**: 登録済みの本の読了日を編集する。
- **アクター**: ユーザー
- **事前条件**: UC3で一覧が表示されていること。
- **事後条件**: 読了日が更新される。
