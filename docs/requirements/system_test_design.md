# システムテスト設計書

## 1. テスト項目
| No | カテゴリ | テスト内容 | 期待値 |
|----|----------|------------|--------|
| 1 | ISBN検索 | 正しいISBNを入力 | 本のタイトル、著者等が表示される |
| 2 | ISBN検索 | 存在しないISBNを入力 | エラーメッセージが表示される |
| 3 | 本登録 | 開始日・終了日を入力して登録 | 一覧画面に正しく表示される |
| 4 | 本登録 | 開始日のみ入力して登録 | 一覧画面に「読みかけ」として表示される |
| 5 | 一覧表示 | 複数冊登録 | 登録した全件が表示される |
| 6 | 編集 | 読了日を後から変更 | 変更が反映される |

## 2. 合否判定基準
- 全てのテスト項目が期待値通りに動作すること。
- 画面遷移がスムーズに行われること。
- API連携で正しい情報が取得できていること。
